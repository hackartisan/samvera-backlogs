script to verify current embargo state values in a given list of works
closes #1248   This task can be used to generate a CSV report to retrieve current embargo values (work_visibility, embargo_release_date, visibility_during_embargo, visibility_after_embargo, embargo_history) for existing works in hyrax given a list of handles with data from dspace.   The output CSV file contains embargo info for the member/file sets in addition to the parent work. These are identified based on the bitstream name in dspace.   The CSV file also includes columns with values in dspace (dspace_handle, dspace_embargo_date, dspace_visibility) and are included to verify that the embargo values were migrated properly.   Usage: RAILS_ENV=production bundle exec rake scholars_archive:embargo_state  Input: The task expects an input file tmp/dspace_bitstream_items_embargo.yml in the format (example): dspace_bitstream_items_embargo: - handle: ""   handle_visibility: "OSU Users"   handle_embargo_date: "06/19/2013"   bitstream:    - bitstream_name: "BiswasSmita2011.pdf"      bitstream_embargo_date: "6/19/13"      bitstream_embargo_visibility: "OSU Users"    - bitstream_name: "license.txt"      bitstream_embargo_date: "6/19/13"      bitstream_embargo_visibility: "OSU Users" - handle: ""   handle_visibility: "OSU Users"   handle_embargo_date: "06/14/2016"   bitstream:    - bitstream_name: "Bremicker.pdf"      bitstream_embargo_date: "6/14/16"      bitstream_embargo_visibility: "OSU Users"    - bitstream_name: "license.txt"      bitstream_embargo_date: "6/14/16"      bitstream_embargo_visibility: "OSU Users"  Output: The rake task generates a CSV file embargo-state-ouput-03-02-2018-08-29-AM.csv that contains the following columns: dspace_handle dspace_embargo_date dspace_visibility work_id url solr_work_visibility solr_embargo_release_date solr_visibility_during_embargo solr_visibility_after_embargo solr_embargo_history
