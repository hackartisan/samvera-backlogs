Human-readable EDTF display transformation
Description Now that dates are stored as edtf strings in the database, we can display them in human readable form in the UI. For example "1984?" becomes "Circa 1984." The actual humanizing functionality comes from duke-libraries/edtf-humanize. Connected to #687 and #1035 Changes  Display transformation for EDTF dates Data Dictionary changes for edtf dates:  change storage from date type to string type remove date index (superseded by #726) add humanize edtf display transformation   Remove Indexing::Dates as it is no longer used (superseded by #726)
