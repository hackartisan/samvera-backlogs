Cannot specify default values in change sets
In DataDictionary::FieldsForChangeSet we ought be able to dynamically assign a default value to a property via the change set: property field.label.parameterize.underscore.to_sym,          multiple: field.multiple?,          type: field.change_set_property_type,          default: field.default_value Yet, this does not seem to work. If we define the property manually, or non-dynamically, such as: property :something,          multiple: true,          default: 'default value' It works fine. So it appears to be somehow related to how were dynamically defining these properties, or that the other options and validations are conflicting with the default value setting process.
