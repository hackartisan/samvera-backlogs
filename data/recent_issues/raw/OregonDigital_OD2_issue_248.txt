[1] Configure proper URL to be sent out when user tries to recover password
Descriptive summary Currently, when a user requests to recover their password, the email that gets sent out includes a url that links to localhost. This is because it is not configured to use the actual url the application is at. This means on staging it should route to od2-staging.library.oregonstate.edu. Once this is configured, this should allow users to request their password on staging. Current functionality When a user requests their password, the link that is sent out goes to localhost Expected functionality When a user requests their password, the link that is sent out goes to od2-staging.library.oregonstate.edu
