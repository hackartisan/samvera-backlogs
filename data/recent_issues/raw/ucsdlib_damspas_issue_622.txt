VRR PDF objects ingested as 'document-source' should link to a generated 'document-service' file in the object interface rather than a zoom
Descriptive summary PDF objects ingested into the DAMS for use in the VRR with file use set as 'document-source', should generate a service PDF that is linked to the image preview in the object interface. Expected behavior Preview image in the object interface should link to a service version of the PDF Actual behavior Preview image tries to link to a zoomable image version of the object which was never generated because it's a PDF Examples https://librarytest.ucsd.edu/dc/object/bd9919328s https://librarytest.ucsd.edu/dc/object/bd4390265d Related work ucsdlib/damsmanager#299 ucsdlib/damsmanager#332
