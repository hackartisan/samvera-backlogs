Download controller isn't properly setting content-disposition header.
Any url that requests a download using the downloads controller (app/controllers/downloads_controller_override.rb)  should generate a response with  a Content-Disposition header of attachment unless you explicitly specify otherwise in the URL parameters. However, this is currently not the case; you get either no Content-Disposition header, or, in the case of an audio file, Content-Disposition: inline; filename="filename.mp3". For instance: $ curl -sD - -o /dev/null  HTTP/1.1 200 OK Date: Mon, 15 Apr 2019 14:12:22 GMT Server: Apache/2.4.29 (Ubuntu) Cache-Control: max-age=0, private, must-revalidate X-XSS-Protection: 1; mode=block X-Request-Id: 71d688ac-7e6d-4790-8607-65af8a3cddb9 X-Frame-Options: SAMEORIGIN X-Runtime: 0.198926 X-Content-Type-Options: nosniff X-Powered-By: Phusion Passenger 5.3.7 Set-Cookie: _chufia_session=a[...]9; path=/; HttpOnly ETag: W/"c[...]6" Status: 200 OK Vary: Accept-Encoding Transfer-Encoding: chunked Content-Type: text/html; charset=utf-8
