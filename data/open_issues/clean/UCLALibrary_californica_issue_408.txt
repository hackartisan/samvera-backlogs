Log when a file is not a TIFF during import
ISSUE During LADNN import, a number of works were created without viable images because the master images were corrupted. Story As a library staff member interested in a complete and swift migration of all content currently maintained in DLCS, I need to know with confidence that all the content I think is being imported into Samvera has been legitimately imported. I need simple text files that allow me to review lists of corrupt files that could not be imported. ACCEPTANCE   Log when characterization fails  Log when characterization yields any mime type other than image/tif  When a failure occurs, the log indicates the work associated with the failing file  When a failure occurs, the log indicates the batch associated with the failing file  Requires #503
