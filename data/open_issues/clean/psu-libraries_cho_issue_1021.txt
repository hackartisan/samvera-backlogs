Enforce application role restrictions via Ability
Roles assigned to a user via fields in a collection govern what abilities that have on that collection and its contained works. Based on a users membership, they should have the following abilities.  collection editor (per-collection role)  manage all metadata on the collection, including access rights and collection submitter role manage all metadata on contained resources add and update resources via CSV add and delete resources via GUI export metadata (for entire collection and individual resources, including restricted objects)   collection submitter (per-collection role)  add resources via GUI add resources via CSV
