Location service returning too much info for some locations
Descriptive summary This is a bug that  found which deals with the location portion of the form. Some locations are returning way too much location information, broader terms all the way out to "Earth" which is so unnecessary. We need to find out why these terms are returning so much information.  Expected behavior When a location is put into the auto complete part of the location field and the form is saved, it should return only a few broader terms up to the country of origin. Actual behavior Some terms are returning all the way up to earth. Steps to reproduce the behavior Create a new work add all required info go to geonames portion of form input portland oregon save the work view the show page.
