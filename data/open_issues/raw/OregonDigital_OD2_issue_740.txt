Homepage accessibility: search box button widget
Descriptive summary The dropdown in the search box (pre-filter for 'all works,' 'osu content,' 'uo content,' etc) does not follow the ARIA menu button pattern Expected behavior Searchbox dropdown has aria-haspopup="true" or aria-haspopup="menu" as per Aria menu button pattern. Related work Audit from #697 Accessibility Concerns Native widgets that act in a different way than their native HTML/Browser implementation are confusing when you can't see or understand any of the visual context provided (shape, CSS, dropdown arrow). Providing hints to screen readers helps to clue the user into what they're interacting with.
